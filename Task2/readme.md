# Task 2: 간단한 웹 게임 만들기 (Copilot Chat – Agent Mode 사용)

## Use case:
- GitHub Copilot Chat의 **Agent Mode**를 활용하여 2048 웹 게임을 만들어 봅니다. Step by step 프롬프팅만으로 Agent가 코드 생성, 파일 생성, 실행까지 자율적으로 수행하는 과정을 체험합니다.

## 목표:
- Copilot Chat의 **Agent Mode**를 활성화하고 사용하는 방법을 익힙니다.
- Agent Mode에 단계적으로 프롬프트를 입력하여 2048 게임을 처음부터 완성합니다.
- Agent Mode가 코드 생성 → 파일 생성 → 실행까지 자율적으로 수행하는 과정을 관찰합니다.
- Copilot Chat의 `Vision` 기능을 활용하여, 디자인 이미지를 기반으로 코드를 제안받아 봅니다.
- Copilot Chat의 `Review` 기능을 활용하여, 코드에 대한 리뷰를 받아 봅니다.

---

## 2048 게임이란?
> 4×4 그리드에서 같은 숫자의 타일을 합쳐 **2048** 타일을 만드는 슬라이딩 퍼즐 게임입니다.
> - 방향키(↑↓←→)로 타일을 이동합니다. 타일은 방향키를 누른 방향으로 쏟아집니다.
> - 같은 숫자의 타일이 충돌하면 합쳐져 두 배가 됩니다.
> - 더 이상 이동할 수 없으면 게임 오버입니다.

---

## Step 1: Agent Mode로 기본 2048 게임 만들기

### 1.1 Agent Mode 시작하기
1. 우측에 열려 있는 **Copilot Chat** 패널을 확인합니다.
   - 이미 채팅창이 열려 있다면, 새로운 대화를 시작합니다.
2. Chat 창 상단의 모드 선택에서 **Agent**를 선택합니다.
   - Ask 모드: 질문에 대한 답변만 제공
   - Plan 모드: 질문에 대해 심도있는 계획을 수립
   - **Agent 모드: 코드 수정, 터미널 실행, 파일 생성 등을 자율적으로 수행**

### 1.2 게임 코드 생성 프롬프트 입력
- Chat 창에 아래 프롬프트를 입력합니다:

```
Task2/src/ 폴더에 2048 게임을 만들어줘.
하나의 index.html 파일에 HTML, CSS, JavaScript 모든 코드를 포함해줘.

요구사항:
- 4x4 그리드 기반
- 방향키(↑↓←→)로 타일 이동
- 타일 이동 및 충돌 시 부드러운 애니메이션 효과 추가
- 같은 숫자 타일이 충돌하면 합산
- 현재 점수 표시
- 게임 오버 판정 (이동 불가 시)
- New Game 버튼으로 재시작
- 타일 숫자별로 다른 배경색 적용

완성되면 python3 -m http.server 8080 으로 서버를 띄워줘.
```

### 1.3 Agent Mode 동작 관찰
- Agent Mode가 자율적으로 수행하는 과정을 관찰합니다:
  1. `Task2/src/index.html` 파일을 자동 생성
  2. HTML 구조, CSS 스타일, JavaScript 게임 로직을 작성
  3. 터미널에서 웹 서버를 실행

- Agent Mode가 제안하는 각 단계에서 **Continue** 또는 **Accept**를 눌러 진행합니다.
  > Agent Mode는 파일 생성이나 터미널 실행 전에 사용자 확인을 요청합니다.

### 1.4 게임 동작 확인
- 브라우저에서 `http://localhost:8080` 으로 접속합니다.
- 방향키를 눌러 타일이 이동하고, 같은 숫자가 합쳐지는지 확인합니다.
- 문제가 있다면 Agent Mode에 바로 수정을 요청합니다:

```
타일이 합쳐질 때 점수가 올라가지 않아. 확인하고 수정해줘.
```

## Step 2: Agent Mode로 게임 기능 확장하기

### 2.1 추가 기능 요청
- 이어서 Agent Mode Chat에 아래 프롬프트를 입력합니다:

```
2048 게임에 다음 기능을 추가해줘:
1. 최고 점수(Best Score)를 localStorage에 저장하고 표시
2. 되돌리기(Undo) 기능 (한 수 되돌리기)
3. 디자인 커스터마이즈해보기
수정이 끝나면 브라우저에서 확인할 수 있게 해줘.
```

- Agent Mode가 `index.html`을 수정하고, 각 기능이 추가되는 과정을 관찰합니다.
- 브라우저를 새로고침하여 새 기능들이 동작하는지 확인합니다.

### 2.2 Vision 기능으로 디자인 개선
- 2048 게임의 원본 디자인 이미지를 검색하여 캡처하거나, 원하는 UI 디자인 스케치를 준비합니다.
- Copilot Chat에 이미지를 **붙여넣기(Ctrl+V)** 하고 아래와 같이 요청합니다:

```
이 이미지와 동일한 디자인으로 2048 게임의 CSS를 수정해줘.
타일 색상, 폰트, 레이아웃을 이미지와 최대한 맞춰줘.
```

- Agent Mode가 CSS를 자동으로 수정합니다.
- 브라우저에서 디자인이 변경되었는지 확인합니다.

---

## Step 3: Agent Mode로 코드 리뷰 및 개선

### 3.1 Agent Mode로 코드 리뷰 요청
- Agent Mode Chat에 아래 프롬프트를 입력합니다:

```
index.html의 JavaScript 코드를 리뷰해줘.
성능, 가독성, 버그 가능성 관점에서 개선할 점을 찾아서 직접 수정해줘.
```

- Agent Mode가 코드를 분석하고, 개선 사항을 직접 적용하는 과정을 관찰합니다.
- 변경 내용을 확인하고 **Accept**를 눌러 적용합니다.


## 지식 확인:
- Copilot Chat의 **Agent Mode**를 활용하여 Step by step 프롬프팅으로 프로젝트를 완성하는 방법
- Agent Mode가 자율적으로 파일 생성 → 코드 작성 → 서버 실행까지 수행하는 워크플로우
- **Vision** 기능으로 이미지 기반 디자인 코드를 생성하는 방법
